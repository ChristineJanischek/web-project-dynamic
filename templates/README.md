# MiFa App‑Repos – Vorlagen

Empfohlene Polyrepo‑Namen:
- `mifa-rideshare`
- `mifa-mindlink`
- `mifa-co2-tracker`

Jedes Repo enthält:
- README mit Ziel, Setup, Aufgaben
- Minimalen WebApp‑Start (HTML/CSS/JS)
- GitHub Actions für Lint/Build (optional)

## Anleitung: neues Repo erstellen
1. Neues leeres GitHub‑Repo anlegen (öffentlich oder privat)
2. Inhalte aus jeweiligem Template kopieren
3. CI optional aktivieren
4. In `version3/aufgabe/index.html` die Links auf das neue Repo/Deployment setzen
